Perguntas a resolver com a ARTE antes da Fase 2
1 — Formato de entrega

O Excel vai ser sempre esta estrutura exacta, ou o formato pode variar entre entregas? Colunas na mesma ordem? Mesmo número de campos?

2 — ID de serviço estável

As colunas 8–11 identificam o serviço por nome por extenso, que pode mudar. É necessário um código estável por serviço que sirva de chave permanente na base de dados.

3 — Chave primária por resposta

Como distinguir uma resposta nova de uma já existente na base de dados? O ficheiro actual não tem um identificador único por resposta que seja fiável entre uploads.

Com estes três pontos resolvidos, a implementação da base de dados pode avançar com confiança total.
Notas e próximos passos
  • Serviço ID unificado: as colunas 8–11 são mutuamente exclusivas mas separadas — é preciso criar uma coluna derivada única antes da ingestão.
  • Canais (col 12) vs. ratings (cols 16–22): a col 12 regista quais os canais usados; as 16–22 avaliam cada canal. Podem coexistir.
  • Campos condicionais (31–34): só existem dados quando col 31 = "Sim". Tratar NaN como "não aplicável", não como dado em falta.
  • Divergência entre ficheiros (col 51): o label difere — confirmar se os dados são comparáveis antes de agregar.
  • Col 52 — Número ID: clarificar finalidade com o cliente. Pode ser ID interno da ARTE.
  • Demográficos (48–51): decisão de produto pendente — podem ficar em standby para a Fase 2.
Mapeamento automático
  • Colunas 8–11: criar coluna derivada service_name — pegar o valor preenchido entre as quatro.
  • Coluna 12: split do texto em colunas booleanas por canal (canal_presencial, canal_digital, canal_app…).
  • Colunas 27–29: índice de transparência por média simples das três sub-dimensões.
  • Colunas 37–39: índice de qualidade do staff por média simples.
  • Colunas 31–34: marcar 32–34 como null quando col 31 = "Não".
  • Coluna 7: mapear texto de entidade para IDs da plataforma.
Precisam do cliente: col 6 (Loja de Cidadão), col 52 (Número ID), cols 48–51 (demográficos).
Impacto na plataforma
  • Bloqueador Cols 8–11: sem mapeamento de serviço, os dados chegam órfãos. Resolver antes de qualquer ingestão.
  • Baixo Cols 27–29 e 37–39: indicadores calculam com menos detalhe mas não quebram.
  • Nenhum Col 12: redundante face às cols 16–22.
  • Nenhum Cols 48–51: irrelevantes na Fase 1.
  • Nenhum Col 52: ignorar até clarificação do cliente.
O único desbloqueador real é o mapeamento das colunas 8–11.
